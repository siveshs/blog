Blockchain Data Analysis

Last edited: Dec 29, 2021

Motivation

  • Which ethereum address has the highest amount of USDC associated with it?
  • What does the distribution of liquidity providers on uniswap look like? How about on sushiswap? And on curve.finance?
  • How has the price for my favorite Art Blocks NFT changed over time?
  • How many people own 0xdeafbeef's NFTs?

So many questions... although there are lots of people writing articles and creating dashboards to track these metrics, I've been curious to learn what it takes to answer some of these questions on my own. After all, isn't the point of a public, decentralized blockchain that me, as a member of the public, can answer any questions that I may have without relying on a central party? Or at least I should be able to verify that the data and analysis done by someone else aligns with how I'd approach the problem?

I'm hoping a side benefit is that this excursion may allow me to better understand and appreciate how various smart contracts and blockchain protocols function under-the-hood.

Current Status

Unsurprisingly, there seem to be many ways to approach this. The tool that I've started experimenting with is Dune Analytics, courtesy some tutorials by Andrew Hong. Eventually, I'd like to learn about The Graph and even further down the road I'd like to see what it takes to query this stuff directly.

dune.xyz seems to have a backend that is continuously adding data from ethereum (and a few other blockchains) into a SQL database that can then be queried using SQL.

So, now I need to learn SQL, dune.xyz and the underlying smart contracts. Sounds like this is going to take some time :-)
