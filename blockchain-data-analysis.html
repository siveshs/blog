Blockchain Data Analysis

Started: Dec 29, 2021

Last edited: Jan 10, 2022

Motivation

  • Which ethereum address has the highest amount of USDC associated with it?
  • How many people own 0xdeafbeef's NFTs?
  • How has the price for my favorite Art Blocks NFT changed over time?
  • What does the distribution of liquidity providers on uniswap look like? How about on sushiswap? And on curve.finance?

So many questions... There seems to be no shortage of people writing articles and creating dashboards to track protocol activity on ethereum. However, I've been curious to learn what it takes to answer some of these questions on my own. After all, part of the point of a transparent, public, decentralized blockchain should be that me, as a member of the public, should be able to answer any questions that I may have without having to rely only on a third party. At the very least, I should be able to verify that the data and analysis done by someone else aligns with how I'd approach the question.

A potential side benefit may be that this excursion will allow me to better understand and appreciate how various smart contracts and blockchain protocols function under-the-hood.

Current Status

Unsurprisingly, there seem to be many ways to approach this. The tool that I've started experimenting with is Dune Analytics, courtesy some fantastic tutorials by Andrew Hong. Eventually, I'd like to learn about The Graph, bitquery, blocknative, setting up a custom infura or alchemy node and even further down the road I'd like to see what it takes to query this stuff directly.

Dune Analytics

dune.xyz seems to have a backend that is continuously adding data from ethereum (and a few other EVM-compatible blockchains) into a postgreSQL database that can be queried using their web interface.

I've never done SQL before starting this project. It's been super fun getting to learn this and being able to see results plotted within Dune for contracts and questions that I care about. Kind of like Excel analysis on steroids.
